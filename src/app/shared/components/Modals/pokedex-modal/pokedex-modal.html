Pokedex – Pokémon Battle Game Full-Stack Game Project

Interactive Pokémon Game

Battle system based on Pokémon abilities and levels

Inspired by the original Pokédex design

Core Game Capabilities

Pokedex es un juego interactivo de colección y exploración de Pokémon construido con Angular y MongoDB, combinando datos en tiempo real de la API de Pokémon con progresión persistente de usuario, mecánicas de filtrado y una UI dinámica.

Pokémon API Integration

Real-time consumption of the public Pokémon API to retrieve Pokémon data, stats, types, and sprites, ensuring up-to-date and accurate information throughout the game.

Advanced Filtering & Search

Dynamic filtering by Pokémon type, name, and attributes, allowing players to quickly explore, compare, and locate Pokémon within the Pokedex interface.

User Profiles & Progression

Persistent user profiles stored in MongoDB, tracking collected Pokémon, player progress, and in-game interactions across sessions.

Interactive Collection System

Players can collect, manage, and review their Pokémon collection through an interactive UI designed to simulate the original Pokédex experience with modern UX patterns.

Scalable MEAN Architecture

Built using Angular for the frontend and MongoDB for persistence, following a modular architecture that supports future features, multiplayer mechanics, or additional game modes.

Responsive & Animated UI

Fully responsive interface with smooth animations and transitions, optimized for desktop and mobile devices to deliver an engaging and immersive gameplay experience.

Architecture Overview

High-level overview of the structural decisions and separation of concerns across the application.

Frontend Architecture
  • Feature-based Angular component architecture
  • Dedicated API service layer for data consumption
  • Modular and reusable UI components
  • Clear separation of presentation, state, and business logic
Backend Architecture
  • RESTful API
  • Mongoose models for Pokémon entities
  • Service-based logic for battles and progression
  • Separation of controllers, services, and data access
